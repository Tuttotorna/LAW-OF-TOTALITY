# Difference Matrix — v0.7.0

## Purpose

This matrix defines the exact difference between Law of Totality and nearby frameworks.

The goal is to avoid both errors:

    overclaiming novelty where existing work already exists;
    underclaiming the compression that the Law of Totality actually provides.

## Minimal Difference

The minimal difference of LOT is:

    object x
    framework F
    actual use U
    valid scope
    excluded critical dependency d
    structural error predicate

This makes it portable across fields that normally use different vocabularies.

## Strongest Conservative Claim

    LOT is not the first framework to notice context, scope, limitations, dependency, or model misuse.
    LOT is a candidate compression of these ideas into a single cross-domain structural-error test.

## Strongest Aggressive But Defensible Claim

    Any finite treatment of an object can be audited by LOT because any finite treatment must use some framework F for some actual use U, and validity depends on whether F preserves the critical dependencies required by U.

## Hard Boundary

LOT does not produce domain expertise by itself.

It cannot decide every critical dependency without domain knowledge.

It provides the slot where the missing dependency must be exposed.

This boundary is essential.
