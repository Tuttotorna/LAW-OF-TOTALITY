# Case Studies — Structural Error Formula

## Purpose

These are not proofs of universality.

They are operational tests.

Each case asks whether the same error-structure appears across domains:

```text
LocalClosureF(x)
∧ ActualUseU(F,x) exceeds ValidScope(F,x)
∧ ∃d [CriticalDepΩ(d,x,U) ∧ ExcludedF(d)]
```

## Case 1 — AI legal answer used as final advice

| Field | Value |
|---|---|
| Object `x` | Legal answer generated by an AI model |
| Framework `F` | Language model trained on broad legal text |
| Use `U` | User treats the answer as jurisdiction-specific legal advice |
| Local closure | The answer is phrased as complete and authoritative |
| Valid scope | General explanation, not binding advice |
| Scope overrun | Used as final decision in a real legal situation |
| Excluded dependency `d` | Jurisdiction, current law, facts of case, lawyer review |
| Structural error | Yes |
| Correction | Mark jurisdiction, date, missing facts, and require qualified review |

Verdict:

```text
ErrΩ(x,F,U) = true
```

## Case 7 — Repository claim without scope conditions

| Field | Value |
|---|---|
| Object `x` | Public repository claim |
| Framework `F` | README, formal note, examples |

| Local closure | Strong words: universal, solves, proves, absolute, final |
| Valid scope | Prototype, conjecture, observational framework, partial formalization |
| Scope overrun | Public claim exceeds actual validation |
| Excluded dependency `d` | Proof status, tests, counterexamples, literature comparison |
| Structural error | Yes if repository omits scope and validation status |

Verdict:

```text
ErrΩ(x,F,U) = true when claim exceeds validation
```

## Pattern found

Across all cases, the error is not merely being incomplete.

The structural error appears when incompleteness is hidden by a local closure and exported into an actual use where the missing dependency matters.
