# Determinability Principle

## 1. Core Statement

~~~text
Nothing can manifest as something without conditions of determinability.
~~~

This is the v0.3.0 hardening of the dependence principle.

The earlier sentence:

~~~text
Everything that exists depends.
~~~

is now treated as compressed language.

The technical form is:

~~~text
Every proper manifestation requires at least one non-zero condition of determinability.
~~~

---

## 2. Formal Core

~~~text
∀x ∈ 𝔻 :
[ManifestΩ(x) ∧ x ≠ Ω]
⇒
∃d [DetCondΩ(d,x)]
~~~

Where:

~~~text
DetCondΩ(d,x)
~~~

means:

~~~text
d is a condition without which x is not determinable as x.
~~~

---

## 3. DepΩ Redefined

~~~text
DepΩ(x) = { d : DetCondΩ(d,x) }
~~~

So:

~~~text
DepΩ(x) ≠ ∅
~~~

means:

~~~text
x has at least one condition through which it can be determined as x.
~~~

---

## 4. Why Determinability Is Stronger Than Generic Dependence

Generic dependence can be too broad.

A critic can ask:

~~~text
Does relation imply dependence?
Does difference imply dependence?
Is every relation a dependency?
~~~

The determinability principle avoids this weakness.

It does not require every relation to be dependence.

It requires only that a manifestation cannot be determinable as itself without at least one determinability-condition.

---

## 5. Minimal Derivation

~~~text
x manifests as x
⇒ x is determinable as x
⇒ x is distinguishable as x
⇒ there is at least one condition of distinction/determinability
⇒ DetCondΩ(d,x)
~~~

Therefore:

~~~text
ManifestΩ(x) ∧ x ≠ Ω
⇒
DepΩ(x) ≠ ∅
~~~

---

## 6. Against Pure Self-Determination

The law rejects pure self-determination of a proper fragment.

Rejected form:

~~~text
∃x :
ManifestΩ(x) ∧ x ≠ Ω ∧ DeterminedOnlyByItself(x)
~~~

Reason:

If x is determined only by itself, without relation, distinction, condition, boundary, field-reference, or determinability-condition, then x cannot be distinguished as x.

Therefore it cannot manifest as a proper x.

---

## 7. Minimal Versus Critical Dependence

For manifestation:

~~~text
MinimalDepΩ(x) ≠ ∅
~~~

For operational error:

~~~text
ExcludedCriticalDepΩ(x,F,U) ≠ ∅
~~~

The first is ontological/structural.

The second is diagnostic/operational.

---

## 8. Core Sentence

~~~text
Existence as a proper manifestation is not first.
Determininability is prior.
Without determinability, there is no manifestation as something.
~~~
