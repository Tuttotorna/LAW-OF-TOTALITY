# Claims Freeze — v1.0.0

## Purpose

This file freezes the public claims for the first external review release.

The purpose is discipline.

The repository is now ready to be attacked by reviewers, not enlarged rhetorically.

## Public Core Claim

The public core claim is:

    local correctness does not imply valid use.

Expanded form:

    A locally correct framework, model, rule, answer, metric, map, label, or theory becomes structurally invalid when it is used beyond its valid scope while excluding a dependency critical to the actual use.

## Formula

    ErrΩ(x,F,U) ⇔
    LocalClosureF(x)
    ∧ ActualUseU(F,x) exceeds ValidScope(F,x)
    ∧ ∃d [CriticalDepΩ(d,x,U) ∧ ExcludedF(d)]

## Allowed Claims

The release may claim:

    this is a candidate structural-error framework;
    this is a pre-review technical object;
    this distinguishes local correctness from valid use;
    this introduces an object-framework-use audit structure;
    this uses excluded critical dependency as the decisive failure condition;
    this includes an Error Atlas with positive cases and negative controls;
    this includes comparative analysis against neighboring frameworks;
    this is ready for hostile review.

## Required Limitation

Every public presentation must include:

    this is not externally validated yet.

## Forbidden Public Framing

Do not frame the project as:

    a completed final doctrine;
    a substitute for domain expertise;
    an automatic detector of every missing dependency;
    a finished scientific settlement;
    a reason to ignore existing fields;
    a public claim of completed validation.

## Strategic Sentence

Use this sentence externally:

    The Structural Error Formula is a candidate audit predicate for detecting when local correctness becomes invalid use.

## Strong But Disciplined Sentence

Use this only after the basic definition is clear:

    Every finite treatment submitted to the formula can be examined in terms of object, framework, actual use, valid scope, and excluded critical dependencies.

## Main Boundary

The formula does not decide all domain facts by itself.

It exposes where domain facts and dependencies must be checked.

This boundary is part of the framework, not an embarrassment.

## Review Position

This is the version to attack.

The goal of v1.0.0 is not praise.

The goal is precise external criticism.
